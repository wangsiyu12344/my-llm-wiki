# HelloAgents フレームワーク

## 概要

HelloAgents は、Datawhale の *Hello Agents* チュートリアルで導入される軽量 AI エージェントフレームワークで、学習者がゼロからエージェントフレームワークを構築するためのガイドです。本番環境向けではなく**教育指向**であり、「すべてはツールである（Everything is a Tool）」というコア哲学を持ちます。

## 設計思想

1. **軽量 + 教育フレンドリー**：コアコードは章ごとに整理され、依存関係は最小限（OpenAI SDK などの必須ライブラリのみ）。学習者は抽象化の迷路に迷わず、具体的なコード行を追跡できます。

2. **標準 API ベース**：OpenAI API 仕様に完全準拠し、移行と統合が容易。ローカルモデルを含む、あらゆる OpenAI 互換サービスを接続可能です。

3. **段階的学習パス**：pip バージョン（例: `hello-agents==0.1.1`）で反復し、各バージョンが学習章に対応。学習者は段階的に深く進めます。

4. **統一ツール抽象化**：Memory、RAG、MCP などの複雑なモジュールをすべて「ツール」として抽象化。抽象化レイヤーを減らし、「エージェントがツールを呼び出してタスクを達成する」というコアロジックに立ち返ります。

## アーキテクチャ

```
hello_agents/
├── core/          # コアフレームワーク層（Agent基底、LLMインターフェース、メッセージ、設定）
├── agents/        # エージェント実装層（Simple, ReAct, Reflection, PlanAndSolve）
└── tools/         # ツールシステム層（基底、レジストリ、非同期実行、組み込みツール）
```

3層分離により、各層を独立して拡張できます。LLMプロバイダーを追加してもエージェントロジックに触れる必要はなく、新しいエージェントパラダイムを追加してもツールシステムを変更する必要はありません。

## なぜ自作するのか？

LangChain のような主流フレームワークには顕著な問題点があります：

- **過剰な抽象化**：概念が多すぎて、単純なタスクでも広範な前提知識が必要
- **API の不安定性**：急速なバージョン反復によりメンテナンスコストが高い
- **ブラックボックス化**：コア実装が不透明で、深いカスタマイズが困難
- **依存関係の複雑さ**：パッケージサイズが大きく、依存関係の競合が頻発

独自フレームワークの構築は、エージェント内部の深い理解、完全な制御、システム設計スキルを提供します。これは特に、金融、医療、教育などの垂直領域で、汎用フレームワークが提供できる柔軟性を超えたカスタマイズが必要な場合に価値があります。
