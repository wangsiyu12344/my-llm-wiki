# LLM 基礎知識

## LLM とは？

大規模言語モデル（LLM）は、ニューラルネットワーク——具体的には Transformer アーキテクチャ——であり、膨大なインターネットデータで事前学習されています。この学習を通じて、パラメータが調整され、シーケンス内の次のトークンを驚くべき精度で予測できるようになります。

**GPT**（Generative Pre-trained Transformer）という名前が本質を表しています：
- **Generative（生成的）**：トークンごとにテキストを生成
- **Pre-trained（事前学習済み）**：特定の用途の前に、インターネット規模のデータで学習済み
- **Transformer**：基盤となるニューラルネットワークアーキテクチャ

LLM は**1トークンずつ**テキストを生成します。各出力は次に来るものの予測です。

## トークン

典型的な英文では：
- 1 トークン ≈ 4 文字
- 1 トークン ≈ 0.75 単語
- 1,000 トークン ≈ 750 単語

## Transformer アーキテクチャ

Transformer は Google チームによって提案され、**LSTM**（Long Short-Term Memory）のような以前のアーキテクチャを置き換えました。LSTM の主な制限は並列化が非常に困難だったことです——処理を順次行う必要がありました。Transformer はこれを解決し、妥当なトークンを生成するだけでなく、驚くべき精度で知性を模倣する大規模な学習を可能にしました。

**パラメータ**は Transformer ニューラルネットワーク内の重みの特定の配置です。より多くのパラメータ（十分なデータと共に）は、一般的により高性能なモデルを意味します。

## LLM の3つのタイプ

1. **Base Model（ベースモデル）**：新しいスキルを学ぶためのファインチューニングに最適。指示チューニングされていない生の事前学習済みモデル。

2. **Chat Model（チャットモデル）**：対話的なユースケースや創造的なコンテンツ生成に最適——例：メール作成、ブレインストーミング、会話。対話用に指示チューニング済み。

3. **Reasoning Model（推論モデル）**：問題解決に最適。多段階推論、数学、プログラミング、論理タスクに最適化されています。

## ステートレス設計

LLM への各呼び出しは**ステートレス**です——モデルは呼び出し間で記憶を持ちません。記憶の錯覚は、毎回入力プロンプトに会話履歴全体を渡すことから生じます。LLM は完全なコンテキストに基づいて、最も可能性の高い次のトークンを予測するだけです。

重要な意味：
- 会話コンテキストが「記憶」である
- 長い会話ほど多くのトークンを消費する
- API 呼び出し間で状態は保持されない

## コンテキストウィンドウ

**コンテキストウィンドウ**は、モデルが次のトークンを生成する際に考慮できる最大トークン数です。これには入力プロンプトと生成出力の両方が含まれます。会話がコンテキストウィンドウを超えると、以前の部分は失われます。
