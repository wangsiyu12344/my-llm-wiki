# エージェント サンドボックス開発

## 概要

サンドボックス開発とは、AI エージェントにホストマシン上で直接コードを実行させるのではなく、隔離されたコンテナ環境内で実行させることを指します。エージェントのアクションは完全に予測可能ではないため、このアプローチは AI エージェント支援開発において特に重要です。サンドボックスが安全境界を提供します。

## サンドボックスの利点

1. **依存関係の隔離**：`pip install` や `npm install` のパッケージがホストサーバーを汚染しません。各プロジェクトの依存関係は完全に独立しています。

2. **安全境界**：エージェントがどのようなコードを実行してもシステムを損傷することはありません。問題のあるコマンドもコンテナ内に制限されます。

3. **クリーンな環境**：プロジェクトを切り替えると完全にクリーンな環境が得られ、「前回の古いバージョンとの競合」問題は発生しません。毎回のセッションが新規に始まります。

## コードの永続化

デフォルトのコンテナは**ステートレス**であり、終了するとすべてのファイルが消えます。サンドボックスで意味のある開発を行うには、コードの永続化を解決する必要があります。

### 方法 A：プロジェクトディレクトリのマウント（推奨）

ホストディレクトリをコンテナ内にマウントし、コンテナ内のファイル変更が実際のディレクトリに直接書き込まれるようにします：

```bash
hermes -p <profile> config set terminal.cwd /home/ubuntu/projects/<project>
```

マウント後、コンテナ内でのファイル変更は実際のディレクトリにリアルタイムで反映され、`git push` はローカルと全く同じように動作します。これが推奨アプローチです。サンドボックスの安全隔離とローカル開発の永続性の両方を得られます。

### 方法 B：純粋サンドボックス + 迅速な push

ディレクトリをマウントしません。コンテナ内でコードを書き、すぐに `commit` + `push` します。コンテナが破棄されるのを待たないでください。欠点：push を忘れたりネットワークが切断されたりすると、コードが失われます。

## まとめ

サンドボックスでの開発はホスト上で直接作業するよりも優れています。方法 A（ディレクトリのマウント）が推奨です。コード作成、テスト、git push のすべてがローカル開発と同じようにスムーズに動作し、サーバーを誤操作から保護します。
